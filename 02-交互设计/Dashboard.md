# 交互设计 — 财富驾驶舱 Dashboard

> **页面路由**：`/`  
> **优先级**：P0  
> **版本**：V1.0

---

## 1. 页面目标

用户打开 App 的第一眼，在 **3 秒内** 回答：

1. 我现在有多少钱？（净资产）
2. 这个月收支如何？（收入/支出/储蓄率）
3. 我的目标进展如何？（目标完成度）

## 3. 组件规格

### 3.1 净资产主卡片（NetWorthHero）

| 属性 | 规格 |
|------|------|
| 高度 | 120px |
| 背景 | 主色 `#1e3a5f`，白色文字 |
| 数字字号 | 36px, font-weight 700, 等宽 |
| 环比 | 16px，绿色↑ / 红色↓ |
| 点击 | 跳转 `/assets` |

**环比计算**：

```
change = (current_net_worth - last_month_net_worth) / last_month_net_worth × 100
```

首月无环比数据时隐藏箭头。

### 3.2 指标卡片（MetricCard）

| 属性 | 规格 |
|------|------|
| 尺寸 | 半宽（桌面两列） |
| 背景 | 白色，border `slate-200` |
| 标签 | 14px, `slate-500` |
| 数值 | 24px, font-weight 600 |
| 链接 | 14px, 主色, "点击查看详情 →" |
| hover | 微阴影 `shadow-md` |

### 3.3 储蓄率条（SavingRateBar）

| 属性 | 规格 |
|------|------|
| 宽度 | 全宽 |
| 进度条颜色 | ≥30% 绿 / 10-30% 黄 / <10% 红 |
| 状态标签 | 健康 ✅ / 一般 ⚠️ / 需关注 ❌ |

### 3.4 目标进度条（GoalProgressItem）

| 属性 | 规格 |
|------|------|
| 高度 | 每项目 56px |
| 图标 | emoji 或 lucide icon |
| 进度条 | 8px 高，主色填充 |
| 百分比 | 进度条右侧 |
| 金额 | 14px, `slate-500` |
| 点击 | 跳转 `/goals` |

## 4. 交互行为

| 动作 | 行为 |
|------|------|
| 点击净资产卡片 | 跳转 `/assets` |
| 点击总资产/总负债 | 跳转 `/assets` |
| 点击本月收入 | 跳转 `/income` |
| 点击本月支出 | 跳转 `/expense` |
| 点击目标进度条 | 跳转 `/goals` |
| 点击「查看全部」 | 跳转 `/goals` |
| 切换月份（P1） | 重新加载当月数据 |

## 6. 数据需求

| 展示项 | 数据来源 | 查询 |
|--------|----------|------|
| 净资产 | assets − liabilities | 聚合 |
| 环比 | 当前 vs 上月净资产 | 需记录历史快照（P1）或从 transactions 推算 |
| 本月收入 | transactions (income) | SUM by month |
| 本月支出 | transactions (expense) | SUM by month |
| 储蓄率 | (income − expense) / income | 计算 |
| 目标进度 | goals | current / target |

**V1 环比方案**：若无历史快照，V1 先隐藏环比，P1 加入 `net_worth_snapshots` 表。

## 7. 验收标准

- [ ] 净资产 = 总资产 − 总负债，精确到分
- [ ] 本月收入/支出与明细页合计一致
- [ ] 储蓄率计算正确（收入为 0 时显示「暂无数据」）
- [ ] 目标进度与 `/goals` 页面一致
- [ ] 所有可点击区域有 hover 态和跳转
- [ ] 空状态有引导 CTA
- [ ] 移动端布局不溢出
